Clustering & Classification Algorithms

1. K-Means Clustering

Goal: Group data into K clusters based on similarity (Euclidean Distance). In the notes, K=3.

The Example Data

PointArea (x)Perimeter (y)Compactness (z)
A15.2614.840.871

C14.2914.090.905

...and so on for D, E, F.

The Algorithm Steps

  1. Initialization: Select 3 random points as "Centroids" (Leaders). Notes selected B, D, and F.
  2. Assignment (Iteration 1): Measure distance from every point to B, D, and F.
    Dist = √((x2-x1)² + (y2-y1)² + (z2-z1)²)
    Assign point to the closest leader.
  3. Update: Calculate the Average of the new groups to find new Centroids.
  4. Repeat: Keep measuring and moving centroids until they stop changing.

2. Hierarchical Clustering

Goal: Build a tree structure (Dendrogram) linking items by distance.

Methods:

  • Single Linkage: Min distance (Nearest Neighbor).
  • Complete Linkage: Max distance (Farthest Neighbor).

3. Decision Trees

Goal: Predict a class label by asking a series of Yes/No questions.

Example (Page 9): Animal Classification.

  • Is it Warm Blooded? → No.
  • Is it Cold Blooded? → Yes.
  • Does it have scales? → Yes.
  • Does it give live birth? → No.
  • Prediction: Reptile.

The Confusion Matrix

Used to check accuracy.

Predicted YES Predicted NO
Actual YES True Positive False Negative (Type II)
Actual NO False Positive (Type I) True Negative

4. COBWEB Clustering

Goal: Incremental clustering for Categorical Data (Non-numeric).

Unlike K-Means (which uses Distance), COBWEB uses Probability.

How it works (from Notes)

1. Start with a Root Node containing all data.

2. Calculate probabilities for every feature.

Example: P(Color = White) = 2/4

3. Split the node if the data isn't "pure" enough.

4. Create a hierarchy of probabilities:

  • Root: 50% White, 50% Black
  • → Child Node 1: 100% White (contains items a, b)
  • → Child Node 2: 100% Black (contains items c, d)
