K-Medoids Clustering (PAM Algorithm)

A clustering algorithm similar to K-Means, but uses actual data points ("Medoids") as centers instead of calculated averages.

1. K-Means vs. K-Medoids

Feature K-Means K-Medoids
Center Mean (Average of points) Medoid (Actual point in dataset)
Distance Euclidean Distance Manhattan Distance
Outliers Sensitive to outliers Robust (Handles outliers well)
Manhattan Distance Formula:
dist = |x₁ - x₂| + |y₁ - y₂|

3. Step 1: Initial Selection (Total Cost = 20)

Randomly select two medoids: C1 = X2 (3,4) and C2 = X8 (7,4).

Distance Calculation (Manhattan)

Calculate distance from every point to C1 and C2. Assign to the closest.

Example for X1 (2,6):
Dist to C1(3,4) = |2-3| + |6-4| = 1 + 2 = 3
Dist to C2(7,4) = |2-7| + |6-4| = 5 + 2 = 7
3 < 7, so X1 belongs to C1.

5. The Decision

Swap Cost (S) = New Total Cost - Previous Total Cost
S = 22 - 20
S = +2

Conclusion:

Since the cost increased (S > 0), swapping C2 with O was NOT a good idea.

We reject the swap and keep the original medoids: C1(3,4) and C2(7,4).
